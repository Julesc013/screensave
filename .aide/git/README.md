# AIDE Git Metadata

This directory stores repo-local Git governance templates, policies, and
report-only detection outputs.

- `commit-template.md` defines the structured commit body expected for
  AIDE-managed work.
- `project-profiles.yaml` defines portable workflow recommendations for AIDE,
  Eureka, Dominium, website, native-client, connector-heavy, data-snapshot, and
  unknown repositories.
- `workflow-detection.json` and `workflow-detection.md` are generated by
  `py -3 .aide/scripts/aide_lite.py git detect` and describe current local Git
  state without fetching, pushing, merging, deleting, or pruning.
- `branch-roles.md`, `promotion-rules.md`, `sync-policy.md`, and
  `prune-policy.md` summarize the corresponding policies for humans.

Q28 is report-only. Q29 owns future helper implementation for land, promote,
sync, and prune operations. No Q28 command mutates branches, remotes, tags,
releases, or GitHub settings.
