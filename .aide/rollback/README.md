# AIDE Rollback Model v0

Q46 defines rollback as observation, ownership inspection, no-apply planning,
dry-run reporting, and validation only.

Rollback plans may describe future restoration or removal candidates when
ownership or prior plan evidence exists. They must preserve target-specific
memory, queues, evidence, golden tasks, manual content, existing tools, and
unknown-ownership paths by default. Q46 never overwrites, deletes, moves,
rewrites, removes managed sections, or applies rollback operations.
