# PL14 Release Notes Draft

## Scope Of This Release

This Plasma release delivers the refactored flagship Plasma saver as a preserved Classic-first product with a truthful lower-band baseline, explicit content and settings systems, real bounded transition support, and product-local BenchLab proof surfaces.

## Notable Preserved Behavior

- Plasma remains `Plasma`.
- `Plasma Classic` remains preserved inside the product.
- Default preset remains `plasma_lava`.
- Default theme remains `plasma_lava`.
- The default runtime path remains the classic raster-plus-flat presentation path.
- The stable baseline remains centered on `gdi` and `gl11`.

## Notable New Capabilities Included

- explicit internal content registry and selection system
- explicit output, treatment, and presentation framework
- optional richer execution lanes for `gl21`, `gl33`, and `gl46`
- one bounded premium dimensional presentation mode: `heightfield`
- bounded real transition support for supported theme, preset, bridge, fallback, and journey cases
- layered Basic, Advanced, and Author-Lab settings surfaces
- product-local BenchLab reporting and bounded forcing support

## Stable Vs Experimental Note

Stable Core Plasma is the preserved Classic/default experience plus the truthful lower-band baseline and its supporting content, settings, and proof surfaces.
Richer lanes, bounded dimensional presentation, transitions, and deeper lab-facing controls ship only as explicit opt-in experimental material in this cut.

## Lower-Band Compatibility Posture

Plasma's stable release posture remains centered on:

- `minimum_kind=gdi`
- `preferred_kind=gl11`
- `quality_class=safe`

Higher lanes remain optional uplifts and are not the hidden minimum.

## Known Limits Summary

- premium-dimensional presentation is currently limited to `heightfield`
- transition proof is bounded to the implemented curated subset
- cross-hardware coverage is still limited to the current validation machine
- numeric performance guarantees are not claimed in this release cut

## Release Tone Boundary

This draft is intentionally factual.
It does not claim more content breadth, more premium presentation breadth, or more validation coverage than the current repo actually proves.
