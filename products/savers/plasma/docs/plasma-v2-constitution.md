# Plasma v2 Constitution

Plasma v2 is a direct-control-first semantic field instrument.

## Identity

Plasma is not a preset picker. Plasma is a deterministic instrument for shaping
living energy fields.

Presets remain useful as examples, bookmarks, migration anchors, and curated
starting points. They must not be hidden masters of runtime behavior once a
direct v2 specification has been resolved.

## Product Center

The product-owned center is:

```text
direct controls
-> bounded semantic spec
-> deterministic compiled plan
-> software reference field
-> material mapping
-> treatment
-> flat presentation
-> existing GDI or optional GL renderer
-> proof bundle
```

The software/reference path is the visual authority. Optional accelerated
realizations must implement the same resolved plan and degrade honestly.

## Compatibility Boundary

The old Plasma engine and preset/theme catalog remain compatibility baseline
and migration evidence. They are not the design center for new Plasma work.

Stable promotion remains blocked while Plasma v2 is in release-candidate hold.
This document does not publish a release, certify compatibility, freeze a public
SDK, or widen Plasma v2 beyond the reviewed candidate scope.
