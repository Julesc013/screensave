# Plasma v2 Release-Candidate Policy

Status: release-candidate policy, not stable release.

Plasma v2 may become a release candidate only after the release-candidate gate
checks proof, package staging, support wording, performance, Manager preview,
Workbench release-readiness, visual review, and native artifact evidence.

The candidate policy keeps these claims separate:

- deterministic reference proof
- accelerated comparison
- performance envelope
- package stage
- support and known limits
- visual review
- release-candidate readiness
- stable release promotion
- compatibility certification

The first admitted candidate identifier is `plasma-v2-rc1`.

## Boundaries

Release-candidate status is not a stable product release.
Package staging is not publication.
Artifact audit is not compatibility certification.
GL11 candidate evidence is not evidence for every accelerated path.
Pack data is not an executable plugin ecosystem.
Manager preview is not install mutation.
Workbench release-readiness is not release publication.

For PAW-H, `stable = false` and release promotion remains blocked.
