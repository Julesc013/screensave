# PX00 Post-PL Program

This document defines Plasma's role in the post-PL program.

## Current Post-PL Posture

Current repo reality is:

- Plasma is already post-`PL14`
- the current ship posture is `GO WITH CAVEATS`
- Stable Core remains Classic-first and lower-band-first
- the stable default center of gravity remains:
  - `default_preset=plasma_lava`
  - `default_theme=plasma_lava`
  - `minimum_kind=gdi`
  - `preferred_kind=gl11`
  - `quality_class=safe`
- `Plasma Classic` remains preserved
- stable versus experimental remains explicit
- requested versus resolved versus degraded truth remains explicit

## Why Plasma Is Now In Post-PL Hardening And Expansion Mode

The bounded `PL00` through `PL14` line already:

- froze Plasma identity
- landed the current engine and lane stack
- landed the current content, transition, settings, and BenchLab seams
- recorded the validation matrix and known limits
- made the current release-cut and ship decision

Later work therefore starts from a real implemented product.
It does not restart a new `PL` planning line.

## PX Role Inside The MX And SY And PX Program

`PX` owns Plasma-local follow-on work.

That includes:

- product proof hardening
- transition hardening
- authoring and provenance
- Lab and validator/comparer growth
- selection intelligence
- output, treatment, field, and dimensional expansion
- product curation and provenance operations
- final product-facing promotion review

`PX` does not own:

- master governance
- queue doctrine
- general platform-law changes
- shared environment or integration contracts that have already crossed the promotion rule

## Current Stable And Experimental And Deferred Stance

### Stable Core

Current stable Core Plasma is the narrow preserved classic/default path:

- preserved `Plasma Classic` identity
- stable built-in classic preset and theme inventory
- `lava_remix` pack compatibility surface
- default `raster` plus `native_raster` plus `flat` path
- truthful `gdi` floor
- truthful `gl11` preferred stable lane
- safe default settings posture with stable-only content filtering and transitions off by default
- product-local BenchLab reporting as support infrastructure

### Experimental But Implemented

The repo currently implements, but does not yet promote as default stable promise:

- `gl21` advanced lane
- `gl33` modern lane
- `gl46` premium lane
- bounded `heightfield` dimensional presentation
- bounded transition subset
- Advanced and Author/Lab settings surfaces
- bounded BenchLab forcing overrides

### Deferred Or Non-Ship

The repo does not currently claim:

- additional dimensional modes beyond `heightfield`
- exhaustive transition pair coverage
- a shipped experimental content pool
- broad cross-hardware lane guarantees
- hard numeric performance SLAs
- scenic or world-simulated premium presentation
- a full unrestricted Plasma debugger

## First-Wave Plasma Priorities

Wave 1 Plasma priorities are driven by the current caveats:

- improve proof posture without widening the stable promise
- harden bounded transition coverage without pretending it is exhaustive
- keep dimensional claims truthful and bounded
- keep BenchLab forcing explicitly bounded and support-facing
- keep stable-versus-experimental messaging tight
- keep the product ledger and risk register current as the canonical PX surfaces

## PX Execution Rule

Post-PL Plasma work should treat the current stable default as protected.

The first goal is to strengthen:

- proof
- control
- authoring discipline
- compositional depth

The first goal is not to broaden stable scope.
