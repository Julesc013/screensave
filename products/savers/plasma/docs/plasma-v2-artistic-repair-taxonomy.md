# Plasma v2 Artistic Repair Taxonomy

This note classifies final stable artistic request-change blockers for Plasma v2.
It is a product-local review aid, not a renderer contract, release publication
packet, compatibility certification, or automatic stable-promotion path.

Plasma v2 remains a direct-control semantic field instrument:

`direct controls -> plasma_v2_spec -> plasma_v2_plan -> plasma_v2_runtime -> field pipeline -> material mapping -> treatment -> presentation -> proof bundle -> review decision`

## Taxonomy

Each recorded request-change blocker must use one or more of these classes:

- `field_shape_quality`: field structure lacks convincing form, continuity, or depth for the accepted scope.
- `motion_quality`: deterministic motion feels wrong for the accepted scope, such as too frantic, too static, or incoherent.
- `material_response`: field-to-color response does not make the selected material distinct enough.
- `luminance_comfort`: brightness or flicker is uncomfortable for the reviewed stable profile.
- `contrast_balance`: contrast is too low for legibility or too high for stable screensaver comfort.
- `treatment_restraint`: post-material treatment is too strong, distracting, or unbounded.
- `control_influence_visible_but_weak`: a direct control is technically visible but not product-legible enough.
- `default_profile_quality`: the default stable profile does not represent Plasma v2 well enough.
- `visualintent_candidate_quality`: VisualIntent candidate evidence is too weak or too broad to support review.
- `gl11_reference_mismatch`: optional GL11 candidate output diverges from the software/reference path without an accepted reason.
- `dark_room_comfort`: quiet-room or low-light review shows unacceptable luminance, contrast, or motion comfort.
- `preview_legibility`: preview or Workbench/Manager review surfaces cannot show the relevant facts clearly.
- `composition_repetition`: field composition repeats or tiles in a way that weakens the instrument identity.
- `artifacting_or_bandings`: visible stepping, banding, seams, or artifacts exceed the accepted scope.
- `documentation_or_evidence_gap`: the decision or evidence packet is missing exact project-owned review facts.

## Boundary

This taxonomy supports review and repair planning only. It does not make proof
hashes, Workbench facts, validators, contact sheets, or external evidence
indexes the owner of final artistic acceptance.
