# Plasma v2 Acceleration Policy

Status: stable-candidate evidence policy.

Plasma v2 keeps the software reference path as the canonical product meaning.
The first acceleration candidate is the GL11 realization/presenter lane, chosen
because U09 already preserved the lower-band posture of `gdi` minimum and
`gl11` preferred.

The candidate is optional. If GL11 is unavailable or not active in a proof
environment, Plasma v2 must fall back to the software reference path and record
that fallback as evidence rather than treating it as a silent success.

PAW-F admits only the first candidate. It does not stabilize GL21, GL33, GL46,
D3D, Vulkan, Metal, premium treatments, or all renderer lanes.

## Required Evidence

- reference-versus-candidate comparison matrix
- mismatch classification
- performance/resource envelope
- second visual review round
- Workbench Profile and Review facts
- development-plane evidence projection as references only

## Claim Boundary

Acceleration evidence can support stable-candidate review. It is not stable release,
final artistic acceptance, compatibility certification, public SDK stability, or
product truth outside ScreenSave review and proof records.
