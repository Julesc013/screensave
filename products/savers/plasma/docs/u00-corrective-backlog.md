# Plasma U00 Corrective Backlog

## Purpose

This note freezes the corrective execution map for Plasma.

It exists to stop future work from treating extra breadth as more important than
coherence, tuning, and proof.

Repo-truth note:
the current snapshot already contains material later-phase salvage work.
This backlog is still the canonical order for Plasma follow-on because it shows
which truths must not be bypassed or regressed.

## Corrective Phase Sequence

| Phase | Focus | Why It Comes Here | Current Snapshot Status | Remaining Obligation |
| --- | --- | --- | --- | --- |
| `U01` | Runtime unification | One runtime had to exist before settings or UI could become truthful. | Landed materially. | Do not reintroduce Classic as a runtime gate. |
| `U02` | Settings authority rewrite | Settings had to own grammar before the UI or presets could be made honest. | Landed materially. | Do not let presets retake hidden authority over output, treatment, or presentation. |
| `U03` | Settings UI redesign | UI only becomes truthful after the authority model is real. | Landed materially. | Keep Basic, Advanced, and Author/Lab boundaries honest and conservative. |
| `U04` | Output, treatment, and presentation rewiring | Distinctions needed to become materially real before stable value could be judged. | Landed materially. | Keep surfaced grammar bounded to supportable paths. |
| `U05` | Destructive simplification | Fake breadth had to be cut before later tuning and promotion work. | Partial and ongoing as doctrine. | Hide or remove unsupported vocabulary and weak surfaces instead of preserving them by inertia. |
| `U06` | Preset and theme rebuild | Content tuning had to happen after the grammar stopped lying. | Landed materially. | Keep stable presets distinct and lower-band-safe. |
| `U07` | Influence and proof harness | Controls must prove they matter before they deserve trust. | Landed materially. | Keep executable dead-setting and near-duplicate checks alive. |
| `U08` | Visual QA bar | Stable and experimental posture needs an explicit acceptance bar. | Landed as doctrine. | Do not widen stable without distinctness and quality evidence. |
| `U09` | Recut and ship posture refresh | Stable versus experimental should be recut only after coherence work, not before. | Landed materially. | Re-run the recut honestly if later work changes the product surface again. |

## Why This Order Is Mandatory

- runtime shape before settings honesty
- settings honesty before UI honesty
- grammar rewiring before preset and theme tuning
- destructive simplification before any renewed breadth claims
- proof before promotion
- recut last

Any future Plasma work that skips this order is likely to start lying again.

## Immediate Blockers For U01-Style Runtime Work

The original U01 blocker set is no longer live in the current snapshot, but it
remains the anti-regression rule for future work:

- no dedicated Classic execution flag or validation gate
- no silent selection rewrite that hides invalid direct requests before the
  selection layer can resolve them honestly
- no runtime story that treats Classic as a second product engine
- no doc or proof note that implies runtime breadth still depends on a Classic
  regime switch

## Immediate Blockers For Settings Honesty

- surfacing unsupported grammar vocabulary as if it were a real option
- letting presets silently override user-selected output, treatment, or
  presentation grammar
- treating transition and curation controls like everyday first-pass visual
  controls
- relying on BenchLab to expose truths the main product surface contradicts

## Immediate Blockers For Visual Distinctness

- accepting close preset pairs as "good enough" without retune or proof review
- overclaiming theme-only separation as if it were independent grammar
- keeping unsupported output, treatment, or presentation vocabulary in
  user-facing stories
- widening stable posture without smoke-backed influence checks,
  preset-signature review, and human visual judgment

## Feature Expansion Freeze

Feature expansion remains frozen behind coherence work.

That means:

- no new broad visual-language family should land before these corrective rules
  are satisfied again
- no new surfaced control should ship unless it is truthful in runtime, UI,
  BenchLab, and proof
- no later release posture should widen stable merely because more enums exist

## What Later Work May Rely On

Later phases may rely on these now-frozen truths:

- `Plasma` identity remains stable
- `Plasma Classic` remains preserved as content identity and compatibility
  surface only
- the stable default center remains `plasma_lava`, `gdi`, `gl11`, and
  `quality_class=safe`
- requested versus resolved versus degraded reporting remains mandatory
- unsupported breadth should be hidden or removed, not narrated into legitimacy

## U00 Conclusion

The corrective series is no longer just a diagnosis of an older broken Plasma.
It is now the rulebook that keeps future Plasma work from drifting back there.
