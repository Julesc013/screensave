# SDK Examples

Purpose: minimal, truthful examples that demonstrate the supported contributor path.
Belongs here: saver-authoring examples, pack manifests, preset files, theme files, and notes that show how to evolve them into real products.
Does not belong here: fake shipping products, private platform experiments, or examples that imply runtime plugin loading.
Type: developer-facing support content.

## Included Examples

- `template_pack/`: minimal pack example targeting the template saver surface with a real `pack.ini`, one preset file, and one theme file
- `../template_saver/`: guide material for the real saver template under `products/savers/_template/`

These examples are intentionally small.
They are meant to be copied, renamed, and adapted into first-class in-repo products or data packs.
