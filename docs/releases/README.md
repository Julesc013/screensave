# Release Docs

Purpose: repo-facing release pages and publication-support notes for tagged ScreenSave releases.
Belongs here: durable release pages, download summaries, and public release copy that points users at the right assets.
Does not belong here: changelog history, implementation notes, or fabricated validation claims.
Type: release-facing documentation.
