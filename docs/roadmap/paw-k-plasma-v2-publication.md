# PAW-K Plasma v2 Publication

Status: blocked by Plasma v2 real-display visual rejection.

PAW-K must not publish Plasma v2 while
`validation/captures/plasma-v2/visual-rejection/verdict.toml` records
`decision = "reject-publication"`.

The PAW-J publication-ready packet remains historical administrative evidence,
not product acceptance. The current product truth is that the tested artifact
failed real-display visual acceptance as low-resolution, blocky, stuttery, and
not visually acceptable. Plasma v2 remains not published.

If a future replacement path receives an accepted real-display visual verdict,
PAW-K may be reconsidered for that accepted path only after final release archive
names, release notes, checksums, provenance, public links, release-page truth,
and a publication receipt are verified.

PAW-K must still preserve these exclusions unless a later reviewed gate changes
them explicitly:

- no compatibility certification broadening
- no all Core saver migration
- no public SDK freeze
- no Manager install/apply mutation beyond its own gate
- no graphical Workbench MVP unless PAW-W1 is opened separately
- no macOS/Linux host claim
- no runtime executable plugin ecosystem

PAW-J created a local publication-prep packet only. PAW-K is the earliest gate
allowed to perform public release publication, but it is currently blocked.

PAW-K0 was the recommended pre-publication closeout gate before the visual
verdict existed. The visual verdict now exists and rejects publication. PAW-K
must not treat publication-ready metadata alone as a substitute for
operator-owned visual acceptance.
